## 🎯 Описание

**Perl Log Analyzer** — это мощный и гибкий инструмент для анализа access-логов веб-серверов в формате Apache/nginx. Проект включает в себя:

- **Генератор тестовых логов** (`generate_logs.pl`) — создает реалистичные логи для тестирования
- **Анализатор логов** (`log_analyzer.pl`) — извлекает статистику и метрики из лог-файлов

## ✨ Возможности

### Генератор логов
- ✅ Генерация 1000+ записей логов за 30 дней
- ✅ Реалистичные IP-адреса (внутренние и публичные)
- ✅ Разнообразные HTTP-методы (GET, POST, PUT, DELETE)
- ✅ Различные эндпоинты (API, статика, админка)
- ✅ Распределение кодов состояния (200, 301, 404, 500)
- ✅ Множество User-Agent строк (браузеры, curl, боты)
- ✅ Переменные размеры ответов

### Анализатор логов
- 🔍 **Топ IP-адресов** по количеству запросов
- 📊 **Статистика HTTP-методов** (GET, POST, PUT, DELETE)
- 📈 **Распределение кодов ответа** (2xx, 3xx, 4xx, 5xx)
- 📉 **Общая статистика** (всего запросов, уникальные IP)
- 💾 **Экспорт в CSV/JSON** (опционально)
- 🚀 **Высокая производительность** даже на больших файлах

## 📝 Формат логов

Анализатор поддерживает стандартный **расширенный общий формат** (common log format):

```
%h %l %u %t \"%r\" %>s %b \"%{Referer}i\" \"%{User-Agent}i\"
```

**Пример записи:**
```
192.168.1.1 - - [10/Nov/2024:07:03:57 +0000] "PUT /images/logo.png HTTP/1.1" 200 1219 "-" "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7) Safari/537.36"
```

**Компоненты:**
- `%h` — IP-адрес клиента
- `%l` — удаленное имя (обычно `-`)
- `%u` — аутентифицированный пользователь
- `%t` — временная метка в формате `[dd/MMM/yyyy:HH:mm:ss +0000]`
- `%r` — строка запроса (`METHOD /path HTTP/version`)
- `%s` — код состояния HTTP
- `%b` — размер ответа в байтах
- `%{Referer}i` — Referer (обычно `-`)
- `%{User-Agent}i` — строка User-Agent

## ⚙️ Настройка

### Настройка генератора логов

Отредактируйте `generate_logs.pl` для изменения параметров:

```perl
# Количество записей
for (1..10000) {  # Измените 1000 на 10000

# IP-адреса
my @ips = ('192.168.1.1', '10.0.0.1', '172.16.0.1', '8.8.8.8', '1.1.1.1');

# Эндпоинты
my @urls = ('/index.html', '/about', '/api/users', '/images/logo.png', 
            '/admin', '/products', '/cart');

# Распределение кодов состояния (больше 200, реже ошибки)
my @statuses = (200, 200, 200, 200, 200, 301, 404, 500);

# Выходной файл
my $output_file = 'access.log';
```
